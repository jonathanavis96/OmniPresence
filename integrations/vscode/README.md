# OmniPresence VS Code Bridge

A minimal VS Code extension that reports workspace, repo, branch, and broad activity to your local [OmniPresence](https://github.com/grafe/OmniPresence) service for Discord Rich Presence.

## Privacy model

| Field | Reported | Condition |
|---|---|---|
| Workspace name | Always | When enabled |
| Repo name | Always | When a git repo is detected |
| Branch name | Always | When a git repo is detected |
| Activity (Editing / Debugging / Running / Idle) | Always | When enabled |
| Exact file name | **Never by default** | Only if `omnipresence.allowFileTitles` is `true` |

All data goes exclusively to `http://127.0.0.1:47831` (your local machine). Nothing is ever sent to an external server.

## Build and install

### Prerequisites

- Node.js 18+ (for native `fetch`)
- npm

### Steps

```bash
cd integrations/vscode
npm install
npm run compile
```

This compiles `src/extension.ts` → `out/extension.js`.

### Run in Extension Development Host

1. Open the `integrations/vscode/` folder in VS Code.
2. Press `F5` (or **Run → Start Debugging**).
3. A new VS Code window opens with the extension loaded.
4. Make sure OmniPresence is running locally — the extension fails silently if it is not.

### Install as a sideloaded extension

```bash
# Install the vsce packaging tool
npm install -g @vscode/vsce

# Package from the integrations/vscode/ directory
vsce package

# Install the generated .vsix
code --install-extension omnipresence-vscode-1.0.0.vsix
```

## Configuration

All settings are under the `omnipresence` namespace in VS Code settings (`Ctrl+,`).

| Setting | Default | Description |
|---|---|---|
| `omnipresence.enabled` | `true` | Enable/disable all reporting |
| `omnipresence.endpoint` | `http://127.0.0.1:47831/integrations/vscode/context` | OmniPresence local endpoint |
| `omnipresence.allowFileTitles` | `false` | Include exact file name in payload |
| `omnipresence.debounceMs` | `2000` | Delay (ms) before sending after an event |

## Payload schema

POST `http://127.0.0.1:47831/integrations/vscode/context`

```json
{
  "source": "vscode",
  "workspace": "RankSentinel",
  "repo": "rank-sentinel",
  "branch": "main",
  "activity": "Editing code",
  "file_title_allowed": false
}
```

With `omnipresence.allowFileTitles` enabled:

```json
{
  "source": "vscode",
  "workspace": "RankSentinel",
  "repo": "rank-sentinel",
  "branch": "main",
  "activity": "Editing code",
  "file_title_allowed": true,
  "file_name": "extension.ts"
}
```

### Activity values

| Value | When |
|---|---|
| `"Editing code"` | An editor is open and focused |
| `"Debugging"` | A debug session is active |
| `"Running"` | A task (build/test/etc.) is executing |
| `"Viewing"` | Viewing a git diff or output panel |
| `"Idle"` | No active editor |

## Git detection

The extension uses the built-in VS Code git extension API first, falling back to reading `.git/HEAD` directly. The repo name is derived from the remote URL (if available) or the workspace folder name.
